# SPRINT PLANNING — Day 5

Date: 9/01/26

## Objectif du sprint

Finaliser le MVP (Minimum Viable Product) en terminant l'interface Web, en implémentant une gestion de campagne simple, et en préparant les éléments pour la présentation finale, y compris une vidéo de démonstration.

## Backlog / Tâches

- [ ] **[US10]** Finaliser l'interface Web :
    - [ ] Implémenter le placement/retrait d'entités via des clics.
    - [ ] Ajouter des boutons pour lancer/arrêter la vague et réinitialiser.
    - [ ] Afficher le résultat de la vague (scoring, héros restants).
- [ ] **[US14]** Implémenter la gestion de la campagne :
    - [ ] Créer une logique qui charge le niveau suivant après une victoire.
    - [ ] Afficher le niveau actuel.
- [ ] **[US18]** Équilibrer les niveaux et le coût des entités.
- [ ] Finaliser et intégrer la fonction de scoring dans les deux UIs.
- [ ] **[US13, US17]** Terminer l'implémentation de l'import/export de donjons si le temps le permet.
- [ ] Préparer la vidéo de démonstration du jeu.
- [ ] Mettre à jour la documentation finale (README, ROADMAP).

## Priorités

1.  **Finalisation de l'interface Web (US10)** : C'est la tâche la plus critique pour avoir un produit présentable.
2.  **Gestion de la campagne (US14)** : Essentiel pour montrer la progression.
3.  **Vidéo de démonstration** : Nécessaire pour la soutenance.
4.  Équilibrage et Scoring : Pour améliorer l'expérience de jeu.
5.  Documentation et Import/Export : À faire en dernier.
